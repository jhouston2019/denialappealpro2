Denial Codes

What Is a RARC Code?

CARC codes tell you why a claim was denied — RARC codes fill in the details the payer left out, from NCCI edit references to specific plan policy citations.

Definition

A Remittance Advice Remark Code — supplemental code that provides additional explanation alongside a CARC code on an EOB.

What a RARC Code Means

RARC stands for Remittance Advice Remark Code. While CARC codes identify the category of adjustment — bundling, authorization, timely filing — RARC codes provide the payer's additional narrative context. They appear on 835 remittance files and patient EOBs, usually as alphanumeric codes starting with letters like M, N, or MA. For example, RARC M15 indicates a payment was reduced because a submitted authorization number was invalid, while N362 points to a specific medical policy the payer applied. There are more than 700 RARC codes in the current code set, though billing teams encounter a smaller subset repeatedly.

Why RARC Codes Matter

Two denials can carry the same CARC code but require completely different appeal strategies depending on the RARC attached. A CO-97 bundling denial with RARC N522 referencing a specific NCCI column 1/column 2 edit pair needs operative documentation proving distinct services. The same CARC 97 with a different RARC might indicate a payer-specific bundling rule instead. RARC codes also help you identify informational messages that do not affect payment — alert RARCs starting with MA are notices only and require no action. Ignoring the RARC means you may appeal the wrong issue and waste a 30-day review cycle.

How RARC Codes Affect Your Claim

RARC codes can confirm whether a denial is appealable or informational. They may reference the exact policy manual section, prior authorization number on file, or claims processing edit that caused the adjustment. On multi-line claims, different line items may share one CARC code but carry different RARC codes — meaning each line needs a tailored response. Payers sometimes add RARC codes after the initial CARC assignment during secondary review, so always compare the RARC on your 835 file to what appears on the portal explanation. A missing or changed RARC can signal the payer adjusted their reasoning between adjudication passes.

Read every RARC code on your denial before writing an appeal. The RARC often names the exact policy, edit, or authorization issue your letter must address — skipping it is the fastest way to a second denial.

What to Do About a RARC Code

Look up each RARC code in the CMS or X12 RARC code list and cross-reference it with the CARC code on the same line item. Determine whether the RARC is informational (MA prefix) or tied to a payment adjustment. Build your appeal argument around both codes — cite the RARC explicitly and attach documentation that refutes or satisfies what it describes. If the RARC references a payer medical policy, obtain that policy document and address each coverage criterion in your letter. Frequently Asked Questions

What is the difference between a CARC code and a RARC code?

A CARC code states the primary adjustment reason, such as bundling or medical necessity. A RARC code adds supplemental detail — for example, which NCCI edit pair triggered the denial or which policy section applies.

Can a denial have a RARC code without a CARC code?

RARC codes are designed to supplement CARC codes, not replace them. You will typically see both on a denied line item. Alert-only RARC codes may appear on paid claims as informational remarks without an associated adjustment.

What does RARC code N130 mean?

RARC N130 is one of the most common remark codes and indicates that consult plan policy provisions were applied. It often accompanies medical necessity or benefit-limit denials and points you to the specific plan language the payer used.

Do I need to address RARC codes in my appeal letter?

Yes, when a RARC code references a specific policy, edit, or documentation requirement. Citing the RARC in your appeal shows the payer you understand their stated reason and are responding to the exact issue they flagged.
